## 1단계 — models.py / parser.py / 단위테스트

### 구현

- **`models.py`**: `SessionState`(str Enum — SLEEP/BLE_ADV/BLE_CONN/OOB_DONE/RANGING/ERR/UNKNOWN),
  `Measurement`(dist_cm/angle_deg/raw/ts), `DeviceEvent`(state/reason/raw/ts),
  **`ParseResult`**(kind/measurement/event/raw/error/ts — CLAUDE.md 확정대로 정의서 9.3의 dict 대신 dataclass)
- **`parser.py`**: 순수 함수 `parse_line(line, ts=None) -> ParseResult`
  - 키 순서·대소문자 무관, `,`/`:` 분해 후 대문자 키 dict화
  - `STATE` 키 → 세션 이벤트(모르는 STATE 값은 `UNKNOWN`으로 살려 전달), `DIST` 키 → 측정
  - 미지 필드(RSSI, Q 등) 무시 = 전방 호환, 깨진 라인은 `kind="invalid"` + 원문·사유 보존(NFR-5)
  - 범위 초과(거리 0~5000cm, 각도 ±90°)는 **파싱 성공** 처리, UI 경고용 헬퍼 `is_out_of_range()` 별도 제공
  - 범위 상수는 파일 상단 상수화(`DIST_MAX_CM = 5000` 등)
- **`requirements.txt`**: 버전 고정 — `flet==0.28.3`, `pyserial==3.5`, `pytest==8.4.1`

## 2단계 — RadarDevice(ABC) + SimulatorDevice

### 구현 (`radar_device.py`)

- **`RadarDevice(ABC)`**: 코딩가이드 3.2 계약 그대로 —
  `connect(port, baud=115200)` / `disconnect()` / `is_connected()` /
  `start_ranging()` / `stop_ranging()` / `list_ports()` +
  콜백 `on_measurement` / `on_state` / `on_log` (기본 no-op 람다)
- **`SimulatorDevice`**:
  - connect 시 데몬 스레드 → `BLE_ADV→BLE_CONN→OOB_DONE→RANGING` 0.5초 간격 방출
  - RANGING 후 거리(20~300cm, ±10)·각도(−90~+90°, ±5) 랜덤 워크 **10Hz**
  - 실패 재현 토글 `simulate_oob_timeout=True` → OOB 단계에서 `STATE:ERR,REASON:OOB_TIMEOUT`
  - 종료: `threading.Event` + `join(timeout=2)` — sleep 중 즉시 깨어남, 좀비 스레드 없음
  - 시뮬 파라미터 전부 상단 상수(`SIM_MEASUREMENT_INTERVAL_S = 0.1` 등)

### 콘솔 데모 결과 (스크래치패드 1회용 스크립트)

- **정상 시나리오**: 세션 4단계가 정확히 0.5초 간격 → 측정 ~100ms 간격 방출.
  `stop_ranging()` 후 1초간 측정 카운트 정지 확인(`18 == 18 → True`), `start_ranging()` 재개,
  `disconnect()` 후 `is_connected=False` (총 23건)
- **실패 시나리오**: `BLE_ADV → BLE_CONN → STATE:ERR,REASON:OOB_TIMEOUT` 후 측정 0건 정지

## 3단계 — UI 전체 (main.py + radar_view.py, 시뮬레이터)

### 구현

- **`radar_view.py` — `RadarView(ft.Container)`**: flet.canvas 직접 렌더 (외부 차트 라이브러리 없음)
  - 동심원 50/100/200/300cm (max_dist 300), 각도선 −90/−45/0/+45/+90°, 0°=화면 위(12시)
  - 좌표 변환: `x = cx + r_px·sin(θ)`, `y = cy − r_px·cos(θ)`, `r_px = dist/max_dist × radius_px`
  - 측정 점 + `"185cm / -10°"` 라벨, 범위 초과 시 주황 경고색, angle=None이면 점 숨김
- **`main.py`** — 정의서 6.1 레이아웃:
  - `USE_SIMULATOR = True/False` 한 줄 전환
  - **연결바**: 포트 드롭다운(+새로고침), Baudrate(기본 115200), OOB 실패 재현 스위치, 상태 LED, 연결/해제 토글
  - **`SessionTimeline`**: 5단계 칩 — 완료=녹 / 진행중=청 / 대기=회 / 실패=적+사유
  - **`NumericPanel`**: 거리·각도·수신율(건/초)·최종수신·세션·상태
  - **`LogConsole`**: timestamp + RX/TX + 원문 + 파싱결과 색상(순수 파서로 재분류), 자동 스크롤, 최대 300줄
  - **동시성**: 콜백은 `queue.Queue`에 put만 → `page.run_thread` 펌프가 **50ms** 주기로 drain 후
    변경 있을 때만 `page.update()` (NFR-1 100ms 이내). UI 코드에 `import serial` 없음 확인(grep).

## 4단계 — QorvoSerialDevice (실물 pyserial)

### 구현

- **`QorvoSerialDevice(RadarDevice)`**:
  - `connect()`: `serial.Serial(port, baud, timeout=1)` + 백그라운드 readline 루프.
    열기 실패 시 예외를 밖으로 던지지 않고 `[LINK]` 접두 로그로 변환 후 복귀
  - **친절한 에러**: 포트 점유(`Access is denied`/`PermissionError`) →
    "포트를 다른 프로그램이 사용 중입니다 (TeraTerm/putty 등을 닫고 재시도)",
    포트 없음 → "포트를 찾을 수 없습니다 (케이블 연결과 포트 이름 확인)"
  - `_reader()`: readline → `on_log`(raw 패스스루) → `parse_line` → kind별 콜백 라우팅.
    케이블 분리 시 `SerialException` 잡아 루프만 종료(앱 무중단, NFR-5)
  - 안전 종료: 플래그 → `join(timeout=2)` → `ser.close()` (readline timeout 1초라 join 항상 성공)
  - `list_ports()`: `serial.tools.list_ports.comports()` 자동 탐색
  - `start_ranging()`/`stop_ranging()`: `START\n`/`STOP\n` 전송, 실패 시 no-op+경고 로그
- **`main.py`**: `USE_SIMULATOR = False` 전환, 포트 없을 때 [연결] 가드,
  연결 실패 시 버튼 미전환, `[LINK]` 로그 → "⚠ 연결 오류" 적색, 실패 재현 스위치는 시뮬 모드에서만 표시

### 검증

- 헤드리스: `COM99` 연결 → 크래시 없이 친절한 에러 로그 / 미연결 `start_ranging` → no-op+로그 /
  미연결 `disconnect` → 안전. (점유 포트 재현은 PC에 실 COM 포트가 없어 건너뜀 — 실보드에서 확인 필요)
- 실물 모드 스크린샷: 포트 목록 비어 있음, [연결] 클릭 시
  "연결할 포트가 없습니다 — 케이블 연결 후 새로고침(↻)을 누르세요" 로그 확인

## 5단계 — 안정성 + 마무리

   사용자가 원한 연결(`_want_connected`)이 끊기면 상태 **끊김(적)** + 로그,
