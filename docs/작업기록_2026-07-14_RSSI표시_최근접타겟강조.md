# 작업 기록 — 2026-07-14

## 작업 목적
- 타겟별 RSSI(신호 세기, dBm) 값을 파싱·표시한다 (요구사항정의서 "확장 여지"로 예정돼 있던 항목).
- 타겟이 여러 개일 때, **거리가 가장 가깝고(동률이면 RSSI가 가장 큰) 타겟**을 빨간색으로 강조 표시한다.

## 변경 내용 요약

### 1) RSSI 값 파싱·저장
- [models.py](../models.py) — `Measurement.rssi_dbm: Optional[float]` 필드 추가.
- [parser.py](../parser.py) — 텍스트 프로토콜(`DIST:85,ANGLE:-12,RSSI:-60`)의 `RSSI` 필드를
  `_to_float()`로 파싱. 숫자가 아니면 `invalid`로 분류(ANGLE과 동일한 엄격도).
  CLI JSON 경로(`_parse_json_line`)는 RSSI 키가 알려져 있지 않아 아직 미지원(향후 과제).
- [radar_device.py](../radar_device.py):
  - `UciSerialDevice._emit_twr_measurement()` — RANGE_DATA NTF의 `meas.rssi`(dBm, 이미 부호
    반영됨)를 `_normalize_rssi()`로 정규화. **UCI 펌웨어에서 rssi=0은 "미지원/측정없음"
    표식**이라 `None`으로 변환(요구사항정의서에 문서화 안 돼 있던 펌웨어 관례 — `uci/qorvo_msg.py`
    주석 `"Minimum rssi or 0 if not supported/available"` 근거).
  - `SimulatorDevice` — 타겟별 RSSI를 −95~−40dBm 범위에서 랜덤 워크로 생성해 보드 없이도
    RSSI 표시를 검증할 수 있게 함.
- [main.py](../main.py) — `TargetRow.rssi_dbm` 추가, 패널 상세 줄이
  `"각도 -12° · RSSI -71.5 dBm · 10 건/초 · 12:00:00"` 형식으로 표시. `rssi_dbm=None`이면
  `"RSSI N/A"`.

### 2) 최근접·최대RSSI 타겟 빨간색 강조
- [main.py](../main.py):
  - `select_primary_target(targets, now, timeout_s)` — 모듈 레벨 순수 함수로 분리(테스트 용이성,
    `parser.py`의 순수 함수 원칙과 동일하게).
    **규칙: 거리(dist_cm) 오름차순 우선, 동률이면 RSSI 내림차순.** 거리 미확보(`dist_cm=None`)
    또는 무수신 타임아웃(2초, `RX_TIMEOUT_S`)을 넘긴 타겟은 후보에서 제외.
  - `COLOR_PRIMARY_TARGET`(`RED_ACCENT_400`) 신설 — 기존 `COLOR_ERR`(경고/에러용, `RED_400`)와
    의미가 겹치지 않게 별도 상수로 분리.
  - `RadarApp._refresh_targets()`가 매 갱신마다 대표 타겟 키를 계산해 `_snapshot_target()`에
    전달 → 그 타겟의 패널 점(●)과 레이더 점/링이 원래 팔레트색 대신 빨간색으로 표시된다.
    원래 팔레트 색 배정(`_target_colors`)은 그대로 유지되므로, 대표 타겟이 바뀌면 이전 타겟은
    자기 색으로 복귀한다(표시 시점의 색 오버라이드일 뿐, 상태는 건드리지 않음).
  - 각도 없는 타겟(UCI 보드, 안테나 1개)도 거리만으로 대표 타겟 후보가 될 수 있다(링 표시로).

### 3) 테스트
- [tests/test_parser.py](../tests/test_parser.py) — RSSI 필드 파싱 3케이스 추가
  (`test_rssi_field`, `test_rssi_missing_defaults_to_none`, `test_rssi_invalid_value`).
  기존 `test_unknown_fields_ignored`는 RSSI가 더 이상 "미지 필드"가 아니므로 예시에서 제외(Q만 유지).
- [tests/test_uci_device.py](../tests/test_uci_device.py) — `make_twr_range_ntf()`에
  `rssi_raw` 파라미터 추가(Q7.1 고정소수, dBm = −raw/2). RSSI 정상값/0(미지원) 케이스 2건 추가.
- [tests/test_target_priority.py](../tests/test_target_priority.py) — 신규 파일.
  `select_primary_target()` 순수 함수 단위테스트 6케이스: 근접 우선, RSSI 동률 처리,
  RSSI 없음 시 우선순위 밀림, 거리 없음 제외, 무수신 제외, 후보 없음.
