## A. 주소(타겟)별 거리 표시 — 핵심 작업

### 1) 데이터 모델에 target_id 추가
- [models.py](../models.py)
  - `Measurement`에 `target_id: Optional[str] = None` 필드 추가 (다중 타겟 식별용).

### 2) 텍스트 프로토콜에 TARGET/ID 필드 지원
- [parser.py](../parser.py)
  - `_parse_measurement()`: `TARGET` 또는 `ID` 키가 있으면 `Measurement.target_id`로 보존한다.
  - `_parse_json_line()` (Qorvo CLI JSON 포맷): 현재는 `target_id=None`으로 고정 — JSON 포맷에서 타겟 식별자를 어디서 가져올지는 미정 (아래 "다음 작업" 참고).

### 3) UCI 실물 디바이스에서 MAC 주소 → target_id 추출
- [radar_device.py](../radar_device.py)
  - `UciSerialDevice`가 RANGE_DATA NTF 처리 시 `meas.mac_add`를 `_normalize_target_id()`로 정규화해 `Measurement.target_id`에 채운다.
  - `_normalize_target_id()`: 소문자화, 빈 값/`"na"`는 `None` 처리, 2바이트(`xx:yy`) 형태면 바이트 순서를 뒤집어(`yy:xx`) 표시용으로 정규화한다.
    - 테스트 기준 예시: 내부 mac_add가 특정 바이트오더일 때 화면 표시값은 `"5f:dd"`.

### 4) UI: 다중 타겟 동시 표시 (최대 5개)
- [main.py](../main.py)
  - `RadarApp`:
    - `_max_targets = 5`, `_latest_targets: dict[str, Measurement]` — target_id별 최신 측정값 보관.
    - `_trim_targets()` — 5개 초과 시 `ts` 기준 오래된 타겟부터 제거.
    - `_target_labels: dict[str, str]` + `target_fields`(타겟 1~5용 `ft.TextField` 5개, `target_wrap`) — 사용자가 각 타겟에 표시 이름(라벨)을 직접 지정할 수 있는 UI. 연결 바에 추가됨.
    - `_on_target_label_commit()` — 라벨 입력 커밋 시 `_target_labels` 갱신.
    - `_target_label(target_id)` — target_id를 사용자가 지정한 라벨로 치환(없으면 target_id 그대로).
    - `_apply_measurement()` — 다중 타겟 측정값을 모아 `radar_view.update_points()`(복수형)로 한 번에 전달. angle이 없는 타겟은 레이더에서 숨기되 패널에는 `target_label: dist` 형태로 표시.
  - `NumericPanel.set_measurement(m, warn, target_label=None)` — target_label(또는 m.target_id)이 있으면 거리 텍스트 앞에 `"라벨: "` 접두어를 붙인다. `dist_cm is None`이면 `"—"` 표시.

### 5) 레이더 뷰: 다중 포인트 렌더링
- [radar_view.py](../radar_view.py)
  - `update_point()` (단수, 포인트 1개) → `update_points()` (복수, `List[Tuple[dist_cm, angle_deg, warn, target_id]]`)로 교체.
  - 각 포인트에 좌표 변환 + 원 + 라벨(`"{target_id}: {dist}cm / {angle}°"`)을 한 번에 그린다.
  - **주의**: `update_point()`를 직접 호출하는 다른 코드가 남아있지 않은지 확인 필요 (현재 `main.py`는 `update_points()`만 호출하도록 이미 수정됨).

### 6) 회귀 테스트 추가
- [tests/test_parser.py](../tests/test_parser.py) — `test_target_id_field`: `"DIST:85,ANGLE:-12,TARGET:2"` → `target_id == "2"`.
- [tests/test_uci_device.py](../tests/test_uci_device.py) — `test_range_data_ntf_uses_mac_address_as_target_id`: RANGE_DATA NTF 주입 시 `target_id == "5f:dd"` 확인.

- API 마이그레이션 (구버전 API가 신버전에서 제거/변경됨):
  - `ft.padding.symmetric(...)` → `ft.Padding(12, 6, 12, 6)` ([main.py](../main.py) `SessionTimeline`)
  - `ft.alignment.center` → `ft.Alignment(0.5, 0.5)` ([main.py](../main.py), [radar_view.py](../radar_view.py))
  - `page.run_thread(app.pump_loop)` (동기 스레드) → `page.run_task(app.pump_loop)` + `pump_loop`를 `async def`로 전환, `time.sleep` → `await asyncio.sleep` ([main.py](../main.py))
- **CLAUDE.md 주의사항**: "기술 스택 고정 — 변경 금지" 항목에 Flet이 명시되어 있다. 이번 변경은 Flet *버전* 업그레이드(라이브러리 자체 교체 아님)이지만, 의도된 변경인지 사용자 확인 없이 진행된 것인지 이 세션 기록만으로는 불명확하다 → **커밋 전 사용자에게 업그레이드 의도를 재확인할 것.**
- `flet run main.py`로 시뮬레이터 기준 정상 기동 여부를 재확인 필요 (이 세션에서 실제 UI 구동 테스트는 수행하지 않음, pytest만 통과 확인).

## C. UCI 벤더 라이브러리 재동기화
`uci/` 폴더 전체에 걸쳐 변경이 있음. 이전 커밋(`ddc819c`, `f8bfed2`)에서는 각 파일 상단에 `Vendored from: https://github.com/sasodoma/uwb-ranging @ commit aad72a0` 주석이 있었는데, 이번 diff에서 **모든 파일에서 이 주석이 제거**됐고 일부 파일은 내용도 바뀌었다. Qorvo 공식 SDK 등 다른 소스로 재동기화한 것으로 추정되나, 세션 기록상 정확한 출처는 확인되지 않는다.

주요 내용 변경:
- [uci/qorvo_cal.py](../uci/qorvo_cal.py)
  - `CalibrationParams`: `PdoaOffs` 제거, `RssiOffSetQ3`(0x4F) / `DebugPllBiasTrim`(0x51, `DebugPllCfg`와 값 중복) / `PdoaIntervalShift`(0x89) 추가.
  - `PhyFrame`의 `sts_len` 비트폭이 4비트 → 8비트로 변경 (`(value >> 16) & (2**4-1)` → `(2**8-1)`).
  - `cal_params_in`: `tx_ant_path`/`rx_ants` 타입이 `Int8`/`Int24` → `Uint8`/`Uint24`로 변경, `pdoa.interval_shift` 항목 추가.
- [uci/qorvo.py](../uci/qorvo.py)
  - `Config`에서 `Traces`(0xA8), `PmMinInactivityS4`(0xA9) 제거 → `ChannelNumber`만 남음.
- [uci/fira_conf.py](../uci/fira_conf.py) — `Config.LowPowerMode` 제거 → `State`만 남음.
- [uci/custom_params.py](../uci/custom_params.py) — 위에서 제거된 `LowPowerMode`/`Traces`/`PmMinInactivityS4`를 `fira.config_params`에서 pop하던 코드 삭제(대상 자체가 없어졌으므로).
- [uci/qorvo_msg.py](../uci/qorvo_msg.py) — `RangingTwrData`/`RangingOwrAoaData`/`RangingOwrUltdoaData`/`RangingOwrDltdoaData`의 `__repr__`에서 `self.nlos.name` → 고정 문자열 `"Unsupported"`로 변경 (nlos 필드가 더 이상 유효하지 않은 것으로 보임).
- [uci/core.py](../uci/core.py) — `Client.close()`를 `try/except`로 방어적으로 변경(트랜스포트 close 실패 시 예외 대신 로깅).
- [uci/addin_transport_uart.py](../uci/addin_transport_uart.py) — `__del__`이 OS 무관하게 방어적 `try/except`로 정리하도록 변경 (기존엔 `os.name != "nt"`일 때만 정리해서 Windows에서 정리 누락 가능성이 있었음).

**주의**: 이 묶음은 "주소별 거리 표시" 작업과 직접 관련이 없다. `radar_device.py`의 `_normalize_target_id()`가 사용하는 `meas.mac_add`가 이 재동기화로 영향받는지는 테스트(`test_range_data_ntf_uses_mac_address_as_target_id`)가 통과하므로 현재는 문제없어 보이나, **커밋 시 A/B/C를 분리할지 하나로 묶을지 사용자 확인이 필요**하다.
